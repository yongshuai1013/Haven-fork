• F-Droid build: the last component still reaching for the wrong tool. The previous two releases pointed most of the Wayland stack at our own wayland-scanner, but wlroots is built from a second place in the script that never got told, so on F-Droid it kept looking for a scanner the build image does not have. Every component is now told once, centrally, so a new one cannot be forgotten. The check for it no longer asks "did it build on my machine" — which could never fail, since that machine has the very tool F-Droid lacks — but asks which tool each component actually used. Nothing changes in the app itself.
